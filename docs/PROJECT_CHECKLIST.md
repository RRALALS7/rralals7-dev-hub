# ✅ Checklist de Projeto Bonito

> Checklist para deixar cada projeto com cara profissional no GitHub.

## README

- [ ] Nome claro
- [ ] Descrição curta
- [ ] Badges
- [ ] Prints ou GIFs
- [ ] Lista de recursos
- [ ] Instalação
- [ ] Como usar
- [ ] Configuração `.env`
- [ ] Roadmap
- [ ] Licença

## Código

- [ ] Estrutura organizada
- [ ] Variáveis bem nomeadas
- [ ] Comentários úteis
- [ ] Tratamento de erro
- [ ] Sem tokens no código
- [ ] `.gitignore`
- [ ] `.env.example`

## Segurança

- [ ] Permissões mínimas
- [ ] Cooldowns
- [ ] Rate limit

- [ ] Logs sem dados sensíveis
- [ ] Dependências revisadas

## Visual

- [ ] Embeds bonitos
- [ ] Mensagens claras
- [ ] Emojis moderados

- [ ] Organização por categoria

## Publicação

- [ ] Testado localmente
- [ ] Commit inicial limpo
- [ ] Releases se necessário
- [ ] Issues ativadas
- [ ] Topics configurados
- [ ] Descrição do repo definida
